# AgriNexus Section-by-Section Video Training Series

Purpose: create clear, non-technical videos that help government leaders, partners, investors, operators, and community users understand what each part of AgriNexus does and how to use it.

Recommended format:
- 8 to 12 short videos instead of one long video.
- Each video should be 2 to 5 minutes.
- Use plain language.
- Start each video with the problem the section solves.
- End each video with the visible outcome or record created.

## Video 1: Platform Opening and Login

Audience: everyone.

Goal: show that AgriNexus is a live operating platform, not a slide deck.

Opening narration:

> Welcome to AgriNexus. This platform brings together learning, workforce development, accessible telehealth, agricultural trade, drone field intelligence, maps, and AI command support into one operating system for rural transformation.

Screen steps:
1. Open the platform URL.
2. Show the login page.
3. Explain that different users can have different roles.
4. Sign in.
5. Pause on the main dashboard.

What to say:

> Once we sign in, the platform opens into a command dashboard. This is where a government partner, project operator, field coordinator, or training leader can see activity across the entire system.

Outcome to highlight:

> The user is now inside a role-based operating platform with connected modules and auditable workflows.

## Video 12: Live Production Setup

Audience: internal launch team, implementation partner, non-technical owner.

Goal: explain how the platform becomes live in a browser through Render.

Opening narration:

> To make AgriNexus live, we deploy the platform to Render, connect a PostgreSQL database, add the provider-engine URL, add the OpenAI key, and validate the production environment.

Screen steps:
1. Show `LIVE_PRODUCTION_ACTIVATION.md`.
2. Explain provider engines first.
3. Explain main platform second.
4. Explain PostgreSQL.
5. Explain OpenAI key.
6. Explain `/status.html`.
7. Explain validation command.

What to say:

> The important point is that AgriNexus does not pretend to be production-ready. It shows exactly what is connected and what still needs credentials.

Outcome to highlight:

> Once the Render variables are complete, AgriNexus becomes accessible from any browser with a live URL.
