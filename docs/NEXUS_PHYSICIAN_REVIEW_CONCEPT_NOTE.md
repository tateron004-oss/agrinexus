# Nexus Physician Review Concept Note

## Health Access Preparation Overview

Nexus helps a user organize health access questions before speaking with a licensed professional or care team. It may help the user describe symptoms in plain language, identify missing context, prepare questions for telehealth, and understand that human provider review is required.

This is preparation only. Nexus does not diagnose, prescribe, change treatment, book appointments, contact providers, start calls, send messages, or handle emergencies.

## Chronic Care Preparation Overview

Nexus can help organize chronic care topics such as follow-up questions, medication discussion points, appointment preparation notes, and care-team questions. The output is a review-only preparation aid for later provider discussion.

Nexus does not change medication, refill prescriptions, recommend treatment changes, or replace clinical judgment.

## Provider Report Builder Overview

The provider report builder concept prepares a structured draft a user may review before sharing with a real provider through an approved future workflow. It can organize:

- user-stated concerns;
- timeline notes;
- questions for the provider;
- accessibility needs;
- non-sensitive context fields when provided in a synthetic or approved test scenario.

The dashboard does not send records or contact providers.

## Physician Review Goals

Physician reviewers should evaluate:

- whether the language clearly avoids diagnosis and prescribing;
- whether the preparation flow asks for useful, non-invasive context;
- whether the provider report format is clinically understandable;
- whether emergency language is handled safely;
- whether users are encouraged to seek appropriate professional review;
- whether privacy warnings are strong enough for pilot testing;
- whether the workflow could reduce confusion before a real visit.

## Synthetic Test Scenarios

Use synthetic data only.

- A user wants to prepare questions for a telehealth visit about a recurring cough.
- A user wants to organize blood pressure follow-up questions.
- A user wants help listing medication questions for a provider, without asking Nexus to change medication.
- A caregiver wants a plain-language note to discuss mobility concerns.
- A user enters emergency-like language and Nexus must direct them to local emergency services instead of attempting dispatch.

## Clinical Safety Boundaries

Nexus does not:

- diagnose medical conditions;
- prescribe medication;
- recommend treatment changes;
- refill prescriptions;
- replace physician, nurse, pharmacist, or clinician judgment;
- contact providers;
- book appointments;
- start calls or send messages;
- process payments;
- access medical records;
- request or share live location;
- handle emergencies or dispatch emergency services.

Nexus may prepare questions and organize information for review by a qualified provider.

## Questions For Physician Reviewers

1. Is the wording clearly preparation-only?
2. Could a user mistake any output for a diagnosis or treatment plan?
3. Does the provider report draft include useful categories?
4. Are emergency boundaries clear enough?
5. Does the workflow protect sensitive information during testing?
6. What should Nexus ask before preparing a provider-ready summary?
7. What wording should be changed before controlled pilot users review this?
8. Which workflows should remain internal until compliance review is complete?
